# Support

Thanks for using Stepwise! Here's how to get help.

## Documentation

- [README](README.md) — overview, quickstart, architecture, and API reference
- [CONTRIBUTING](CONTRIBUTING.md) — local setup, tests, lint, and build
- [docs/hyde.md](docs/hyde.md) — deep dive on the HyDE retrieval approach
- Interactive API schema at `/docs` when the API is running

## Questions & discussion

- Browse [existing issues](https://github.com/Padraigobrien08/MultiModal_Rag/issues)
  first — your question may already be answered.
- For usage questions, open a
  [GitHub Discussion](https://github.com/Padraigobrien08/MultiModal_Rag/discussions)
  if enabled, otherwise a regular issue.

## Reporting bugs & requesting features

Use the [issue templates](https://github.com/Padraigobrien08/MultiModal_Rag/issues/new/choose):

- **Bug report** — something isn't working as documented.
- **Feature request** — an idea or improvement.

Please include enough detail to reproduce (version/commit, steps, logs).

## Security issues

Do **not** report security vulnerabilities in public issues. Follow the process
in [SECURITY.md](SECURITY.md).

## Response expectations

Stepwise is a community-maintained open-source project. Maintainers respond on a
best-effort basis — there is no guaranteed response time or SLA.
